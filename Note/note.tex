\section{ATLAS latest study}% (fold)
\label{sec:atlas_latest_study}

    The couplings of longitudinally polarized $W$ and $Z$ bosons to the Higgs boson prevent the divergence of tree-level Vector Boson Scattering (VBS) amplitudes at high energies when the two outgoing bosons are longitudinally polarized, restoring the unitarity at the TeV scale. Thus, the VBS processes with longitudinal vector bosons provide a sensitive test of the electroweak (EW) symmetry-breaking mechanism.

    The most recent ATLAS study is detailed in Ref.~\cite{ATLAS:2025wuw}. This study focuses on VBS. Specifically, it investigates the EW production of same-sign $W$ boson pairs (EW $W^{\pm} W^{\pm} jj$), a process to which VBS provides a significant contribution. The key concepts and event selection criteria of this work are summarized below.

    The object selection criteria are as follows:
    \begin{itemize}
        \item \textbf{Electrons:} Must have transverse momentum $p_{\text{T}} > 27~\mathrm{GeV}$ and pseudorapidity $\abs{\eta} < 2.47$, excluding the calorimeter transition region of $1.37 < \abs{\eta} < 1.52$.
        \item \textbf{Muons:} Must have $p_{\text{T}} > 27~\mathrm{GeV}$ and $\abs{\eta} < 2.5$.
        \item \textbf{Jets:} At least two jets with $\abs{\eta} < 4.5$ are required in each event. The leading jet must have $p_{\text{T}} > 65~\mathrm{GeV}$, and the subleading jet must have $p_{\text{T}} > 35~\mathrm{GeV}$.
    \end{itemize}

    The Signal Region (SR) event selection requires:
    \begin{itemize}
        \item A same-sign lepton pair with an invariant mass $m_{\ell\ell} > 20~\mathrm{GeV}$.
        \item Missing transverse momentum magnitude, $E_{\text{T}}^{\text{miss}}$, greater than $30~\mathrm{GeV}$.
        \item The two highest-$p_{\text{T}}$ jets must satisfy a dijet invariant mass $m_{jj} > 500~\mathrm{GeV}$ with an absolute rapidity difference $\abs{\Delta y_{jj}} > 2.0$.
    \end{itemize}

\section{Monte Carlo simulation}% (fold)
\label{sec:monte_carlo_simulation}

    \subsection{Sample Generation}% (fold)
    \label{sub:sample_generation}
        The signal processes are the electroweak (EW) production of $W^{\pm}W^{\pm}jj$ in various polarization states. We simulate the EW same-sign $W$ boson pair production at a center-of-mass energy of $\sqrt{s} = 13~\text{TeV}$. The events are generated using \verb|MadGraph 3.3.1|~\cite{Alwall:2014hca}. Both $W$ bosons are forced to decay leptonically. Parton showering and hadronization are simulated with \verb|Pythia 8.306|~\cite{Sjostrand:2014zea}, followed by a fast detector simulation using \verb|Delphes 3.4.2|~\cite{deFavereau:2013fsa}. Jets are reconstructed via the anti-$k_t$ algorithm~\cite{Cacciari:2008gp} with a distance parameter of $R = 0.4$, implemented in \verb|FastJet 3.3.2|~\cite{Cacciari:2011ma}. Reconstructed jets are required to have a transverse momentum of $p_{\text{T}} > 25~\text{GeV}$.

        The following \verb|MadGraph| scripts are used to generate the Monte Carlo (MC) samples for these polarized processes:

        \paragraph{Longitudinal mode: $pp \to W^{\pm}_{\text{L}}W^{\pm}_{\text{L}} jj$}
        \begin{lstlisting}
            generate p p > j j w+{0} w+{0} QCD=0, w+ > l+ vl
            add process p p > j j w-{0} w-{0} QCD=0, w- > l- vl~
            output EW_WWjj_LL
            launch EW_WWjj_LL

            shower=Pythia8
            detector=Delphes
            analysis=OFF
            madspin=OFF
            done

            Cards/delphes_card.dat

            set run_card nevents 10000
            set run_card ebeam1 6500.0
            set run_card ebeam2 6500.0

            set run_card ptj 34.0
            set run_card ptl 26.0
            set run_card misset 29.0

            set run_card etaj 4.6
            set run_card etal 2.6

            set run_card mmjj 480
            set run_card deltaeta 1.9

            set run_card ptj1min 64
            set run_card ptj2min 34

            set run_card use_syst False

            done
        \end{lstlisting}

        \paragraph{Transverse mode: $pp \to W^{\pm}_{\text{T}}W^{\pm}_{\text{T}} jj$}
        \begin{lstlisting}
            generate p p > j j w+{T} w+{T} QCD=0, w+ > l+ vl
            add process p p > j j w-{T} w-{T} QCD=0, w- > l- vl~
            ...
        \end{lstlisting}

        \paragraph{Mixed mode: $pp \to W^{\pm}_{\text{L}}W^{\pm}_{\text{T}} jj$}
        \begin{lstlisting}
            generate p p > j j w+{0} w+{T} QCD=0, w+ > l+ vl
            add process p p > j j w-{0} w-{T} QCD=0, w- > l- vl~
            ...
        \end{lstlisting}

    \subsection{Event Selection}% (fold)
    \label{sub:event_selection}

        The event selection criteria applied after the \verb|Delphes| simulation are defined as follows:
        \begin{itemize}
            \item \textbf{Lepton cut:} The invariant mass of the same-sign lepton pair, $m_{\ell\ell}$, must be greater than $20~\mathrm{GeV}$.
            \item \textbf{MET cut:} The missing transverse momentum magnitude, $E_{\text{T}}^{\text{miss}}$, must be greater than $30~\mathrm{GeV}$.
            \item \textbf{Jet cut:} The two highest-$p_{\text{T}}$ tagging jets must satisfy a dijet invariant mass of $m_{jj} > 500~\mathrm{GeV}$ with an absolute rapidity difference of $\abs{\Delta y_{jj}} > 2.0$.
        \end{itemize}

    \subsection{\texorpdfstring{$WW$}{WW} Center-of-Mass Reference Frame}% (fold)
    \label{sub:ww_center_of_mass_reference_frame}

        To properly study the scattering dynamics, we modify our \verb|MadGraph| scripts to define the $W$ boson polarizations in the $W^{\pm}W^{\pm}$ center-of-mass (CM) frame.

        The following generation commands are used to produce the MC samples with this specific reference frame definition:

        \paragraph{Longitudinal mode: $pp \to W^{\pm}_{\text{L}}W^{\pm}_{\text{L}} jj$}
        \begin{lstlisting}
            generate p p > w+{0} w+{0} j j QCD=0, w+ > l+ vl
            add process p p > w-{0} w-{0} j j QCD=0, w- > l- vl~
            output MG5/EW_WWjj_LL-WW_cmf
            launch MG5/EW_WWjj_LL-WW_cmf

            shower=Pythia8
            detector=Delphes
            analysis=OFF
            madspin=OFF
            done

            Cards/delphes_card.dat

            set run_card nevents 10000
            set run_card ebeam1 6500.0
            set run_card ebeam2 6500.0

            set run_card ptj 34.0
            set run_card etaj 4.6
            set run_card mmjj 480
            set run_card deltaeta 1.9

            set run_card etal 2.6
            set run_card ptl 26.0
            set run_card misset 29.0

            set run_card ptj1min 64
            set run_card ptj2min 34

            set run_card me_frame 3, 4, 5, 6

            set run_card use_syst False

            done
        \end{lstlisting}

        \paragraph{Transverse mode: $pp \to W^{\pm}_{\text{T}}W^{\pm}_{\text{T}} jj$}
        \begin{lstlisting}
            generate p p > w+{T} w+{T} j j QCD=0, w+ > l+ vl
            add process p p > w-{T} w-{T} j j QCD=0, w- > l- vl~
            ...
        \end{lstlisting}

        \paragraph{Mixed mode: $pp \to W^{\pm}_{\text{L}}W^{\pm}_{\text{T}} jj$}
        \begin{lstlisting}
            generate p p > w+{0} w+{T} j j QCD=0, w+ > l+ vl
            add process p p > w-{0} w-{T} j j QCD=0, w- > l- vl~
            ...
        \end{lstlisting}

        The key modification lies in the \verb|me_frame| setting. More details can be found in ref.~\cite{BuarqueFranzosi:2019boy}. It is crucial to note that when utilizing the in-line decay syntax, \verb|MadGraph| expands the decay products directly into the final-state particle list. Consequently, the indices must be specified as \verb|3, 4, 5, 6| to encompass all the decay products (leptons and neutrinos) required to reconstruct the $WW$ system.

        The remaining shower, detector, and run card settings for the transverse and mixed modes are identical to those of the longitudinal mode and are omitted here for brevity.

    \subsection{Simulation with Sherpa}% (fold)
    \label{sub:simulation_with_sherpa}

        To investigate the cross-section discrepancy, we employ a different Monte Carlo generator, \verb|Sherpa 3|~\cite{Sherpa:2024mfk}, which is also the primary generator utilized in the ATLAS analysis.

        We simulate the EW same-sign $W$ boson pair production at a center-of-mass energy of $\sqrt{s} = 13$ TeV. Both $W$ bosons are forced to decay leptonically. Parton showering and hadronization are handled internally by \verb|Sherpa|, followed by a fast detector simulation using \verb|Delphes 3.4.2|~\cite{deFavereau:2013fsa}. Jets are reconstructed via the anti-$k_t$ algorithm~\cite{Cacciari:2008gp} with a distance parameter of $R = 0.4$, implemented in \verb|FastJet 3.3.2|~\cite{Cacciari:2011ma}. The reconstructed jets are required to have a transverse momentum of $p_{\text{T}} > 25$ GeV.

        Unlike \verb|MadGraph|, which can generate pure polarized samples, \verb|Sherpa| generates events and assigns polarization weights to each of them. These weights represent the probability of an event belonging to a specific polarization state. Due to technical constraints within the generator, the $W^{+}W^{+}jj$ and $W^{-}W^{-}jj$ processes must be generated as separate samples.

        The following YAML configuration snippet illustrates the generation of the $W^{+}W^{+}jj$ sample using \verb|Sherpa|:
        \paragraph{$W^{+}W^{+} jj$ sample}
        \begin{lstlisting}
            EVENTS: 100
            RANDOM_SEED: 1

            BEAMS: 2212
            BEAM_ENERGIES: 6500

            PDF_SET: NNPDF30_nlo_as_0118

            ME_GENERATORS:
              - Comix
            COMIX_DEFAULT_GAUGE: 0

            # scales
            MEPS:
              CORE_SCALE: VAR{0.5*Abs2(p[2]+p[3])}

            # physical constants and width settings
            PARTICLE_DATA:
              24: {Width: 0}   # W Boson
              23: {Width: 0}   # Z Boson
            WIDTH_SCHEME: Fixed

            # decay and polarization settings
            HARD_DECAYS:
              Enabled: true
              Channels:
                24,12,-11: {Status: 2}   # W+ -> nu_e e+
                24,14,-13: {Status: 2}   # W+ -> nu_mu mu+
              Pol_Cross_Section:
                Enabled: true
                Reference_System: [Lab, COM]

            # process definition: same-sign W boson pair production with two jets
            PROCESSES:
              - 93 93 -> 24 24 93 93:
                  Order: {QCD: 0, EW: 4}

            # final state particles cuts
            SELECTORS:
              - [PT, 93, 34, 10000]
              - [Eta, 93, -4.6, 4.6]
              - [Mass, 93, 93, 480, 10000]
              - [DY, 93, 93, 1.9, 10000]

              - VariableSelector:
                  Variable: PT
                  Flavs: 93
                  Ranges:
                    - [60.0, 10000.0]
                  Ordering: [PT_UP]

            # output settings
            EVENT_OUTPUT: HepMC3_GenEvent[EW_WpWpjj.hepmc]
        \end{lstlisting}

        \paragraph{$W^{-}W^{-} jj$ sample}
        \begin{lstlisting}
            EVENTS: 100
            RANDOM_SEED: 1

            BEAM_ENERGIES: 6500

            COMIX_DEFAULT_GAUGE: 0

            # scales
            MEPS:

            PARTICLE_DATA:
              24: {Width: 0}   # W Boson

              Enabled: true

                -24,-12,11: {Status: 2}   # W- -> nubar e-
                -24,-14,13: {Status: 2}   # W- -> nubar mu-

    \subsection{Particle-Level Selection}% (fold)
    \label{sub:particle_level_selection}

        Upon a detailed review of the reference thesis~\cite{stange2025longitudinal}, we identified that their reported yields and cross-sections are evaluated at the particle level rather than the reconstructed level. To perform a fair comparison, we modified our event selection framework to operate on particle-level objects.

        Specifically, the variables accessed in the \verb|Delphes| analysis were updated as follows:
        \begin{itemize}
            \item \textbf{Leptons:} Reconstructed leptons (\verb|Electron| and \verb|Muon| collections) are replaced by generator-level stable final-state leptons from the \verb|Particle| branch.
            \item \textbf{Missing Transverse Momentum:} The reconstructed missing transverse energy (\verb|MissingET|) is replaced by the true missing transverse momentum (\verb|GenMissingET|).
            \item \textbf{Jets:} Reconstructed jets (\verb|Jet|) are replaced by truth-level jets (\verb|GenJet|).
        \end{itemize}

        Furthermore, the kinematic selection criteria at the particle level are different from the reconstructed-level SR selection defined in Section~\ref{sub:event_selection}. Based on the reference thesis, the particle-level selection is defined as follows:
        \begin{itemize}
            \item \textbf{Leptons:} Exactly two same-sign leptons with $p_{\text{T}} > 27~\mathrm{GeV}$. Muons must satisfy $\abs{\eta} < 2.5$. Electrons must satisfy $\abs{\eta} < 2.47$, excluding the transition region $1.37 \leq \abs{\eta} \leq 1.52$, with an additional restriction of $\abs{\eta} < 1.37$ for the $ee$ channel.

            \item \textbf{Dilepton Mass \& $Z$-veto:} The dilepton invariant mass must be $m_{\ell\ell} > 20~\mathrm{GeV}$. Additionally, a $Z$-boson veto is applied to the $ee$ channel, requiring $\abs{m_{ee} - m_{Z}} > 15~\mathrm{GeV}$ to suppress background contributions.

            \item \textbf{Missing Transverse Momentum:} $E_{\text{T}}^{\text{miss}} \geq 30~\mathrm{GeV}$.

            \item \textbf{Jets:} At least two jets are required. The leading and subleading jets must satisfy $p_{\text{T}} > 65~\mathrm{GeV}$ and $p_{\text{T}} > 35~\mathrm{GeV}$, respectively. The dijet invariant mass threshold is relaxed to $m_{jj} \geq 200~\mathrm{GeV}$, and the rapidity difference must be $\Delta y_{jj} > 2$.
        \end{itemize}

        The PDF sets are also updated to the same as the reference thesis~\cite{stange2025longitudinal}. We use the \verb|NNPDF23_nlo_as_0119| PDF set~\cite{Ball:2012cx} for \verb|MadGraph| and \verb|PDF4LHC21_40_pdfas| PDF set~\cite{PDF4LHCWorkingGroup:2022cjn}for \verb|Sherpa|.

    \subsection{Background Processes}% (fold)
\label{sub:background_processes}

    In this section, we focus on the primary background sources: the irreducible QCD $W^\pm W^\pm jj$ production and the reducible $W^{\pm}Zjj$ process.

        \subsubsection{QCD \texorpdfstring{$W^{\pm}W^{\pm}jj$}{WWjj} Production}% (fold)
        \label{subs:qcd_wwjj_production}

            The QCD-induced same-sign $W$ boson pair production in association with two jets is an irreducible background, as its final state is identical to that of the signal. At tree level, this process is of order $\mathcal{O}(\alpha_S^2 \alpha^4)$.

            The following \verb|MadGraph| scripts are used to generate the MC samples:
            \begin{lstlisting}
                generate p p > w+ w+ j j QCD<=2 QED<=2, w+ > l+ vl
                add process p p > w- w- j j QCD<=2 QED<=2, w- > l- vl~

                ...
            \end{lstlisting}

        % subsubsection qcd_wwjj_production (end)

        \subsubsection{INT \texorpdfstring{$W^{\pm}W^{\pm}jj$}{WWjj} Production}% (fold)
        \label{subs:int_wwjj_production}

            The interference (INT) $W^{\pm}W^{\pm}jj$ background arises from the quantum interference between the purely electroweak and the QCD-induced amplitudes. At tree level, this process is of order $\mathcal{O}(\alpha_S \alpha^5)$.

        % subsubsection int_wwjj_production (end)

        \subsubsection{\texorpdfstring{$W^{\pm}Zjj$}{WZjj} Production}% (fold)
        \label{subs:wzjj_production}

            The $WZ$ production in association with jets, where $WZ \to \ell\nu\ell\ell$, is the most prominent reducible background. It mimics the same-sign $WW$ topology when one of the leptons from the $Z$ boson decay is ``lost'' (e.g., falls outside the detector acceptance or fails identification criteria).

            The following \verb|MadGraph| scripts are used to generate the MC samples:
            \begin{lstlisting}
                generate p p > w+ z j j QCD=0, w+ > l+ vl, z > l+ l-
                add process p p > w- z j j QCD=0, w- > l- vl~, z > l+ l-

                ...
            \end{lstlisting}

        % subsubsection wzjj_production (end)

        \subsubsection{Non-Prompt Leptons}% (fold)
        \label{subs:non_prompt_lepton}

            Leptons originating from hadron decays and jets misidentified as leptons are referred to as non-prompt leptons. Since this background is estimated using data-driven methods in the experimental analysis, we omit its estimation in our current Monte Carlo simulation framework.

        % subsubsection non_prompt_lepton (end)

        \subsubsection{Full Process vs. Narrow Width Approximation (NWA)}% (fold)
        \label{subs:full_process_and_nwa}

            The most rigorous approach to generating these events is to simulate the full $2 \to 6$ process. For instance, for the EW $W^{\pm}W^{\pm}jj$ signal, the full process command would be:
            \begin{lstlisting}
                generate p p > j j l+ vl l+ vl QCD=0
                add process p p > j j l- vl~ l- vl~ QCD=0
            \end{lstlisting}

            However, simulating the full process requires evaluating a vast number of Feynman diagrams and integrating over a highly complex phase space. This often leads to severe integration inefficiencies and divergences, making it prohibitive to generate a sufficient number of events.

            We investigated the $W^{\pm}Zjj$ case and found that the full simulation and the Narrow Width Approximation (NWA) approach produce highly consistent absolute cross-sections after applying the SR cuts. Therefore, we temporarily adopt the NWA for our current generation setup to ensure efficiency.

    \subsection{Determination of the Cross-Section for Sherpa Samples}% (fold)
    \label{sub:what_cross_section_should_be_used_in_sherpa_samples}

        When utilizing \verb|Sherpa| to generate MC samples, there are two primary stages. First, the generator performs a numerical integration over the defined phase space to calculate the total cross-section. Subsequently, it generates events and writes the kinematic information and local cross-section into the resulting \verb|HepMC| file.

        In our previous analyses, the cross-section was estimated by extracting the values recorded event-by-event in the \verb|HepMC| file and computing their average. However, the values written to the event record are local estimates derived during the generating stage. Consequently, they are subject to large statistical fluctuations, which can introduce biases into the final normalization.

        Conversely, the cross-section reported in the \verb|Sherpa| log output that was evaluated during the initial phase-space integration stage is more suitable. Because this integration stage utilizes vast sampling points before generating events, it reduces statistical fluctuations and yields a more stable cross-section.

    \subsection{MadGraph Run Card Selection Criteria Issues}% (fold)
    \label{sub:madgraph_run_card_selection_criteria_issue}

        To increase the event generation efficiency for the SR phase space, we applied kinematic cuts at the generator level (as detailed in Section~\ref{sub:ww_center_of_mass_reference_frame}). While these cuts are appropriate for generating pure $W^{\pm}W^{\pm}jj$ signal samples, they introduce biases when extending the analysis to Control Regions (CR) or simulating reducible backgrounds such as $W^{\pm}Zjj$.

        For the CR, the analysis investigates the low-$m_{jj}$ region, specifically $200~\mathrm{GeV} < m_{jj} < 500~\mathrm{GeV}$. The previous generator-level cut of \verb|set run_card mmjj 480| is problematic, as it truncates almost all valid events that would populate this CR.

        Furthermore, for the $W^{\pm}Zjj \to \ell\nu\ell\ell$ background, the global lepton transverse momentum cut \verb|set run_card ptl 26.0| causes a underestimation. The \verb|ptl| parameter in \verb|MadGraph| enforces a minimum $p_{\text{T}}$ requirement on \textit{all} generated leptons.

        \textbf{Example:} Consider a $W^+Z \to \ell^+ \nu \ell^+ \ell^-$ event where the two same-sign leptons have $p_{\text{T}} = 45~\mathrm{GeV}$ and $35~\mathrm{GeV}$, while the third lepton ($\ell^-$) is soft with $p_{\text{T}} = 15~\mathrm{GeV}$. In the detector simulation, this soft $\ell^-$ might be ``lost'' (e.g., falling outside the acceptance or failing identification), leaving exactly two same-sign leptons that pass the SR cuts ($p_{\text{T}} > 27~\mathrm{GeV}$). However, because the third lepton's $p_{\text{T}}$ is lower than the $26.0~\mathrm{GeV}$ threshold, \verb|MadGraph| will discard this event at the generation level. Consequently, this topology is completely lost.

        To resolve these issues, the \verb|MadGraph| run card scripts must be updated to use looser, ordered cuts. Taking the QCD $W^{\pm}Zjj$ process as an example:

        \begin{lstlisting}
            generate p p > w+ z j j QCD<=2 QED<=2, w+ > l+ vl, z > l+ l-
            add process p p > w- z j j QCD<=2 QED<=2, w- > l- vl~, z > l+ l-
            output MG5/WZjj_QCD-NWA-v370
            launch MG5/WZjj_QCD-NWA-v370

            shower=Pythia8
            detector=Delphes
            analysis=OFF
            madspin=OFF
            done

            Cards/delphes_card.dat

            set run_card nevents 10000
            set run_card ebeam1 6500.0
            set run_card ebeam2 6500.0

            set run_card pdlabel lhapdf
            set run_card lhaid 260000

            set run_card ptj 20.0
            set run_card etaj 5.0
            set run_card mmjj 150.0
            set run_card deltaeta 1.5
            set run_card ptj1min 50.0
            set run_card ptj2min 25.0

            set run_card etal 2.7
            set run_card ptl1min 20.0
            set run_card ptl2min 20.0

            set run_card misset 10.0

            set run_card use_syst False
            set run_card cut_decays True

            done
        \end{lstlisting}

        The primary changes include relaxing the dijet invariant mass baseline to \verb|mmjj 150.0|, and replacing the global \verb|ptl| cut with ordered lepton cuts: \verb|ptl1min 20.0| and \verb|ptl2min 20.0|. This ensures that only the leading and subleading leptons are constrained, leaving the third lepton unrestricted. Other kinematic variables are relaxed to safely encompass the full reconstructed-level phase space.

\section{Dense Neural Network}% (fold)
\label{sec:dense_neural_network}

    Because the kinematic relationships between the signal and background processes are complex, a single variable is insufficient for optimal separation. Therefore, multivariate analysis techniques, specifically Deep Neural Networks (DNNs), are employed to extract sensitive features to distinguish signal and background events~\cite{ATLAS:2025wuw}. We consider the Dense Neural Neworks (Dense-NNs) as our first trial.

    \subsection{Input Features}% (fold)
    \label{sub:input_features}

        The Dense-NN relies on a set of low-level and high-level kinematic variables. These input features are defined as follows:
        \begin{itemize}
            \item $(p_{\text{T}}^{l}, \eta^{l}, \text{flavor}^{l})$: Transverse momentum, pseudorapidity, and flavor type of the leading and sub-leading leptons.

            \item $(p_{\text{T}}^{j}, \eta^{j})$: Transverse momentum and pseudorapidity of the leading and sub-leading jets.

            \item $E_{\text{T}}^{\text{miss}}$: Missing transverse momentum magnitude.

            \item $\Delta\phi(x, \ell_1)$: Difference in azimuthal angle between the object $x$ and the leading lepton. The object $x$ can be the sub-leading lepton, leading jet, sub-leading jet, or the missing transverse momentum $E_{\text{T}}^{\text{miss}}$.

            \item $Z_{\ell}^{\star}$: Zeppenfeld variables~\cite{Rainwater:1996ud} of the leading and sub-leading leptons. The pseudorapidities of the leptons and tagging jets are used to define this variable:
                \begin{equation}
                    Z_{\ell}^{\star} = \abs{\frac{\eta_\ell - \frac{1}{2} \left( \eta_{j_1} + \eta_{j_2} \right) }{\eta_{j_1} - \eta_{j_2}}}.
                \end{equation}

            \item $m_\text{T}(x, E_{\text{T}}^{\text{miss}})$: Transverse mass of the object $x$ and the $E_{\text{T}}^{\text{miss}}$ system. The object $x$ can be the leading lepton, sub-leading lepton, or the di-lepton system. The standard definition for a two-object system is:
                \begin{equation}
                    m_\text{T}(x, E_{\text{T}}^{\text{miss}}) = \sqrt{\left( E_{\text{T}}^{x} + E_{\text{T}}^{\text{miss} } \right)^2 - \left| \vec{p}_{\text{T}}^{x} + \vec{p}_{\text{T}}^{\text{miss}} \right|^2},
                \end{equation}
                where the scalar transverse energy is evaluated as $E_{\text{T}} = \sqrt{m^2 + |\vec{p}_{\text{T}}|^2}$.

            \item $m_{\text{T}}^{0}$: Projected transverse mass:
                \begin{equation}
                    m_{\text{T}}^{0} = \sqrt{\left( p_{\text{T}}^{\ell_1} + p_{\text{T}}^{\ell_2} + E_{\text{T}}^{\text{miss} } \right)^2 - \left| \vec{p}_{\text{T}}^{\ell_1} + \vec{p}_{\text{T}}^{\ell_2} + \vec{p}_{\text{T}}^{\text{miss}} \right|^2}.
                \end{equation}
                Because the rest mass of an electron or muon is negligible in this high-energy context ($m \approx 0$), the transverse energy of a single lepton simplifies to its transverse momentum ($E_{\text{T}}^{\ell} \approx p_{\text{T}}^{\ell}$). The key difference between the standard di-lepton transverse mass ($m_{\text{T}}$) and the projected transverse mass ($m_{\text{T}}^{0}$) lies in how the scalar energy sum is constructed. The standard $m_{\text{T}}$ treats the two leptons as a single composite system, whereas $m_{\text{T}}^{0}$ treats the two leptons as separate massless entities and directly sums their individual scalar transverse momenta.

            \item $(\Delta R_{\ell\ell}, \Delta\eta_{\ell\ell}, m_{\ell\ell}, p_{\text{T}}^{\ell\ell})$: Angular distance $\Delta R$, pseudorapidity difference, invariant mass, and transverse momentum of the di-lepton system.

            \item $(\Delta R_{jj}, \Delta y_{jj}, m_{jj}, \Delta\phi_{jj})$: Angular distance $\Delta R$, rapidity difference, invariant mass, and azimuthal angle difference of the di-jet system.

            \item $\displaystyle \frac{p_{\text{T}}^{\ell_1} p_{\text{T}}^{\ell_2}}{p_{\text{T}}^{j_1} p_{\text{T}}^{j_2}}$: Ratio of the scalar transverse momenta products of the leptons to the jets.

            \item $\min(\Delta R_{\ell, j})$: Minimum angular distance $\Delta R$ between any of the signal leptons and jets.
        \end{itemize}

    % subsection input_features (end)

    \subsection{Dataset Construction and Validation Strategy}% (fold)
    \label{sub:datasets_construction}

        To ensure the robustness of the neural network models and prevent over-training, a 5-fold cross-validation strategy is employed in the ATLAS analysis. The total dataset is separating into training, validation, and test sets with a fixed ratio of 3:1:1.

        The assignment of each event to a specific subset is determined by its event number. For a given fold index $i_{\text{fold}} \in \{0, 1, 2, 3, 4\}$, the splitting rule is defined as follows:
        \begin{itemize}
            \item \textbf{Test Set}: Events satisfying $(\text{Event Number} - i_{\text{fold}}) \pmod 5 = 0$.
            \item \textbf{Validation Set}: Events satisfying $(\text{Event Number} - i_{\text{fold}} + 1) \pmod 5 = 0$.
            \item \textbf{Training Set}: All remaining events that do not fall into the test or validation sets.
        \end{itemize}

        The training processes are divided into two distinct stages: a primary inclusive network ($\text{DNN}_{W^{\pm}W^{\pm}}$) designed to separate the EW signal from all backgrounds, and secondary polarization-specific networks ($\text{DNN}_{\text{pol}}$) trained exclusively on the EW samples to distinguish specific polarization states.

    \subsection{Kinematic Distributions}% (fold)
    \label{sub:kinematic_distributions}

        To illustrate the separation power of the input features, we present the normalized distributions of several key kinematic variables. The comparisons are divided into two categories, reflecting the two-stage strategy of the Dense-NN training: (1) separating the $W^{\pm}W^{\pm}jj$-EW signal from various background processes, and (2) distinguishing between the different polarization states of the $W^{\pm}W^{\pm}jj$ electroweak production.


        \subsubsection{Signal vs Background Discrimination}% (fold)
        \label{subs:signal_vs_background_discrimination}
            In the first stage of the DNN, the goal is to isolate the inclusive $W^{\pm}W^{\pm}jj$-EW signal from the dominant backgrounds, which include $W^{\pm}W^{\pm}jj$-QCD, $W^{\pm}Zjj$-EW, and $W^{\pm}Zjj$-QCD.

            Figure~\ref{fig:dist_bkg} shows the distributions of six features. The VBS signature is characterized by two forward jets with a large invariant mass ($m_{jj}$) and a large pseudorapidity gap ($\abs{\Delta\eta_{jj}}$), making them powerful discriminants against QCD-induced processes. Furthermore, the Zeppenfeld variable ($Z_{\ell_1}^{\star}$) quantifies the centrality of the lepton relative to the tagging jets; signal leptons tend to fall between the two forward jets. General kinematics, such as the leading jet transverse momentum ($p_{\text{T}}^{j_1}$), di-lepton invariant mass ($m_{\ell\ell}$), and missing transverse momentum ($E_{\text{T}}^{\text{miss}}$), provide additional discrimination power against the $WZ$ and QCD backgrounds.

    \subsection{Neural Network Architecture}% (fold)
    \label{sub:neural_network_architecture}

        The Dense-NN architecture utilized in the analysis is presented in Figure~\ref{fig:dnn_architecture}. 

        The network begins with an input normalization layer. This layer applies the scaling to the input features first. Then, normalizing the resulting distribution to have a mean of 0 and a variance of 1.

        Following the nomalization layer, the architecture relies on a core block consisting of a fully-connected (dense) layer, a Batch Normalization layer, a Swish~\cite{DBLP:journals/corr/abs-1710-05941} activation function, and a Dropout layer for regularization. This composite block is repeated $L$ times to form the deep hidden layers of the network.

        Finally, the network terminates with an output dense layer.
        \begin{figure}[htbp]
            \centering
            \begin{tikzpicture}[
                node distance=1.2cm,
                % 定義方塊樣式，文字旋轉 90 度
                box/.style={rectangle, draw=black, thick, minimum width=1.2cm, minimum height=3.5cm, align=center, fill=gray!10, font=\small},
                block/.style={rectangle, draw=black, thick, minimum width=1.0cm, minimum height=3.0cm, align=center, fill=white, font=\small},
                arrow/.style={thick, -{Stealth[scale=1.2]}}
            ]

            % 1. 輸入與常態化層
            \node (input) [box, fill=green!5] {\rotatebox{90}{Input}};
            \node (norm) [box, right=of input] {\rotatebox{90}{Normalization}};

            % 2. 隱藏層區塊 (重複單元)
            \node (dense) [block, right=of norm] {\rotatebox{90}{Dense}};
            \node (bn) [block, right=0.4cm of dense] {\rotatebox{90}{Batch Norm}};
            \node (swish) [block, right=0.4cm of bn] {\rotatebox{90}{Swish}};
            \node (dropout) [block, right=0.4cm of swish] {\rotatebox{90}{Dropout}};

            % 使用虛線框住重複區塊
            \node[draw=blue, dashed, inner xsep=0.3cm, inner ysep=0.3cm, fit=(dense) (dropout)] (hidden_block) {};
            \node[anchor=south, color=blue] at (hidden_block.north) {Repeated $L$ times};

            % 3. 輸出層
            \node (output) [box, fill=red!5, right=of hidden_block] {\rotatebox{90}{Output}};

            % 連接箭頭
            \draw [arrow] (input) -- (norm);
            \draw [arrow] (norm) -- (hidden_block.west |- dense);
            \draw [arrow] (hidden_block.east |- dense) -- (output);

            % 區塊內部連接箭頭
            \draw [arrow] (dense) -- (bn);
            \draw [arrow] (bn) -- (swish);
            \draw [arrow] (swish) -- (dropout);

            \end{tikzpicture}
            \caption{Schematic representation of the feed-forward neural network architecture. The sequence begins with an input normalization layer. The core architecture consists of $L$ repeated hidden layers, each comprising a dense layer with a constant number of neurons, followed by batch normalization, a Swish activation function, and a dropout layer for regularization. The final dense layer provides the binary classification output.}
            \label{fig:dnn_architecture}
        \end{figure}

        The Swish activation function utilized in the hidden blocks is defined as:
        \begin{equation}
            \mathrm{swish}_{\beta}(x) = \frac{x}{1 + \euler^{-\beta x}}
        \end{equation}
        where $\beta$ is a learnable parameter or a fixed constant. Unlike ReLU, Swish is smooth and non-monotonic, characteristics that have been shown to improve gradient propagation and performance in deep networks.
